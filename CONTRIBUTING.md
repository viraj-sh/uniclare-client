## üêû Reporting Bugs

To ensure bugs are handled efficiently:

1. **Search existing issues** to avoid duplicates.
2. Use the official **Bug Report** template:
   *Issues ‚Üí New Issue ‚Üí Bug Report*
3. Provide all required details:

   * Affected component
     (API / MCP / Frontend / Docker / Deployment)
   * Installation method
     (local clone, Dockerfile build, Docker Hub image, cloud deployment, etc.)
   * Version, branch, or commit hash
   * Steps to reproduce
   * Expected vs. actual behavior
   * Logs or stack traces
   * Environment information

Clear, complete reports help significantly with debugging and faster resolution.

---

## ‚ú® Submitting Feature Requests

Feature proposals are welcome.
Use the **Feature Request** template and include:

* The feature summary
* The problem it solves
* Expected behavior or proposed solution
* Any parity notes with the official *Uniclare APP* or *Student Portal" client
* Additional technical or usage context

This keeps feature planning structured and actionable.

---

## ‚ùì Questions & Discussions

If you need clarification or want to discuss ideas, open a **Question/Discussion** issue.
Keeping all discussions in Issues ensures information is organized and accessible to all contributors.
